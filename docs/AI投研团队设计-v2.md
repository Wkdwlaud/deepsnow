## 0. 设计原则

整个系统建立在七条不可妥协的原则之上。后续所有具体设计都是这七条的展开。

1. **找好公司和定仓位是两件事**。物理隔离——研究公司的人/Agent 不看市场季节，判断市场季节的人/Agent 不看具体公司。两条线在最后由人合成。
2. **Agent 的定位是"内部卖方"，不是决策者**。Agent 输出研究报告、跟踪报告、池子维护结果，最终决策由人在会议机制里完成。
3. **股票池是核心物理产出**。所有研究的沉淀，都必须落到股票池这个分层、动态、可审计的数据库里。没有进入股票池的研究等于没做。
4. **方法论必须嵌入 Prompt**。把吴伟志的语言（四季、树粮菜、GREAT、买断公司标准）作为强制框架写进每个 Agent 的 Prompt，让方法论物化在系统里，不依赖你每次记得用。
5. **复盘是唯一的进化通道**。所有决策、依据、结果都要归档。季度复盘比任何单次研究都重要。
6. **估值方法论必须闭环隔离**。成长股用成长的估值工具（PEG、未来市值法），价值股用价值的估值工具（PE/PB/股息率）。绝不允许"用尺子称重量"——用 PE 否定高增长成长股，或用"未来空间大"买入高估值价值股。每个研究组的 Prompt 中硬编码本组的估值红线。
7. **建仓纪律必须硬编码**。左侧 30% → 底部确认 50% → 右侧打满 100%。这不是建议，是不可逾越的纪律。协调 Agent 必须在每次调仓建议中校验是否符合建仓协议。

## 2. 投资研究组设计

### 2.1 五个行业组的划分依据

按吴伟志公开提到的"投资大模型"思路，分五个组：

| 组别 | 覆盖范围 | 核心矛盾 | 主要估值方法 | 估值红线 |
|------|---------|---------|------------|---------|
| 消费组 | 食品饮料、家电、纺服、社服、轻工、农林牧渔 | 品牌、渠道、消费力周期 | PEG、DCF、PS（新消费） | 禁止以"PE高"否定高增长标的 |
| 医药组 | 创新药、CXO、医疗器械、医疗服务、中药、药店、疫苗、血制品 | 研发管线、政策、出海 | PEG、管线 NPV、PS（未盈利 Biotech） | 禁止以"PE高"否定管线价值，必须用 NPV |
| 科技组 | 半导体、消费电子、计算机、传媒、通信、AI 应用、商业航天 | 技术拐点、产业周期、海外映射 | PEG、PS、未来市值法 | 禁止以"没有利润"否定早期公司，必须用未来市值法 |
| 周期组 | 有色、化工、钢铁、煤炭、机械、电新（光伏锂电）、军工、地产链 | 供给侧、库存周期、商品价格 | PB、EV/EBITDA、重置成本 | 注意：周期股 PE 最低时往往是卖点，PE 最高时往往是买点 |
| 高股息组 | 银行、保险、公用事业、运营商、能源央企、交运 | 分红可持续性、利率环境、资产负债表 | 股息率、PE、PB、DDM | 禁止以"未来空间大"买入高估值标的，必须满足低 PE + 低 PB + 高股息率 |

> **估值隔离原则**（源自吴伟志访谈）：
> - 成长组（消费/医药/科技）的好股票，在价值组看来是"毒药"——这是正常的，不是问题
> - 价值组（高股息）的好股票，在成长组看来"没空间"——这也是正常的
> - 两边互相"敌视"才说明各自的闭环是对的。如果成长组和价值组互相认可同一只股票，反而要警惕方法论出了问题

> 备注：行业归属不是严格互斥的，比如"创新药 + AI 医疗"涉及医药组和科技组。处理方式是**主归属唯一，副标签可叠加**，跨组协作通过协调 Agent 处理。

### 2.2 估值方法论详解

#### 成长组通用估值工具箱

**工具一：PEG（有利润的成长股）**

适用：已有稳定利润的传统行业成长股

```
PEG = PE / 未来3年预期利润增速(%)
- PEG < 1：偏低估
- PEG 0.5-0.8：较好的买入区间
- PEG > 1.5：偏贵，除非增速还在加速
```

> 吴伟志举例：40倍 PE + 100%增速 → PEG=0.4（便宜）；20倍 PE + 10%增速 → PEG=2.0（贵）。不能只看 PE 高低。

**工具二：未来市值法（新经济/早期公司）**

适用：无利润或早期的新经济公司

```
未来市值 = 行业规模 × 公司份额 × 利润率 × 合理 PE

示例：
- 行业未来规模：1万亿
- 公司可能份额：50% → 5000亿收入
- 行业利润率：10% → 500亿利润
- 给20倍PE → 1万亿市值
- 当前市值1000亿 → 10倍空间 → 不贵
```

> 吴伟志原话："未来有1万亿市值，你有十倍的空间，那就不贵。"

**工具三：PS（辅助）**

适用：高增长但利润率尚未稳定的公司，作为辅助参考

#### 价值组估值工具箱

**核心三角：低 PE + 低 PB + 高股息率**

```
买断公司标准（价值股）：
- PE ≤ 10倍：2-3年PE就相当于买了整个公司两三年回本
- PB ≤ 1倍：以低于净资产的价格买
- 股息率 ≥ 4-5%：即使股价不涨，每年也有可观回报
- 三项至少满足两项

吴伟志原话："PE只有两三倍，相当于买完这个公司两三年就给我回本了。"
```

**价值陷阱识别**：

高股息只是三年五年，未来就没有了 → 价值陷阱，不碰

```
价值陷阱检查清单：
- 股息来源是否可持续（经营现金流 vs 一次性收入）
- 行业是否在结构性衰退（不只是周期性低谷）
- 资产负债表是否健康（分红不能靠借债）
```

**价值投资的止损原则**：原则上不止损。因为价值投资永远是逆势投资，如果买入后止损，那是趋势投资的思维。卖出只因为：（1）发现是价值陷阱；（2）别的地方有更好的机会需要调配资金。

#### 周期组特殊估值逻辑

```
周期股估值陷阱：
- PE 最低时（盈利顶峰）→ 往往是卖点
- PE 最高时（盈利低谷）→ 往往是买点
- 核心看供需关系拐点，不是看利润水平
```

### 2.3 每个行业 Agent 的产出物

每个组的产出按时间频度分三档：

**A. 深度报告**（季度 1-2 篇）
- 触发条件：新公司入池、重大事件、季度复盘
- 产出格式：详见第 9 章"报告模板"
- 必须包含：GREAT 五维分析、树/粮/菜定性、价格-价值反应状态、目标价区间、风险点

**B. 跟踪报告**（月度 + 事件触发）
- 月度：股票池每家公司的关键指标更新、估值变化、机构动向
- 事件触发：业绩披露、重大公告、政策变动、行业数据出炉

**C. 行业框架文档**（持续迭代）
- 这是该组的"小模型"物理形态
- 包含：行业图谱、商业模式、关键指标、政策敏感点、历史周期复盘、龙头清单
- 每季度迭代一次

### 2.4 行业组的输入数据源

| 数据类型 | 来源 | 用途 |
|---------|------|-----|
| 财务数据 | 上市公司公告、年报季报 | 财务分析、估值 |
| 行业数据 | 行业协会、统计局、海关、Wind/同花顺 | 景气跟踪 |
| 政策信息 | 国务院、部委、监管机构官网 | 政策周期 |
| 卖方研报 | 各大券商研究所 PDF | 观点参考（不照搬，70分深度） |
| 产业新闻 | 财联社、华尔街见闻、36 氪 | 事件跟踪 |
| 公司公告 | 巨潮资讯、港交所披露易 | 实时事件 |
| 调研纪要 | 业绩会、机构调研、雪球 | 管理层判断 |

> 重要：Agent 引用任何数据**必须给出原始来源链接或文件位置**。不能审计的报告等于不能用。
>
> 深度标准（源自吴伟志访谈）：卖方研究是 70 分深度，我们要 95 分。Agent 必须在报告末尾列出"还需要进一步研究的问题"——这些就是从 70 分到 95 分需要人工补充的部分（专家访谈、实地调研、用户体验）。

## 3. 投委会设计

投委会的核心是**只看市场、不看公司**。它的存在意义是把"现在该几成仓"和"该买哪家"彻底分开，避免在选股时被情绪干扰、在定仓位时被持仓污染。

### 3.1 季节判断 Agent

**输入指标体系**（四组共 16 项核心指标）：

**估值组**（权重 25%）：
- 万得全 A PE / PB 历史分位（近 10 年）
- 沪深 300 PE / 股债收益差
- 创业板指 PE 分位
- 恒生指数 / 恒生科技 PE 分位

**资金组**（权重 25%）：
- 融资融券余额及环比
- 新基金发行规模（月度）
- 北向资金累计净流入（90 日滚动）
- ETF 申赎情况

**情绪组**（权重 25%）：
- 全市场日均成交额
- 换手率（流通市值口径）
- 涨停家数 / 跌停家数比
- 全市场破净率

**基本面组**（权重 25%）：
- 制造业 PMI
- 社融同比增速
- PPI / CPI 同比
- 上市公司盈利预期变化（一致预期 EPS 修正）

**输出格式**（月度报告）：

```markdown
# 市场季节判断报告 - YYYY 年 MM 月

## 一、综合判断
当前季节：[春初 / 春中 / 春末 / 夏初 / 夏中 / 夏末 / 秋初 / 秋中 / 秋末 / 冬初 / 冬中 / 冬末]
建议仓位区间：XX% - XX%
相比上月：[升温 / 持平 / 降温]

## 二、四组指标评分
- 估值组：X / 10 分（依据：...）
- 资金组：X / 10 分（依据：...）
- 情绪组：X / 10 分（依据：...）
- 基本面组：X / 10 分（依据：...）

## 三、关键观察
- 本月最重要的 3 个市场变化
- 季节切换的早期信号（如有）

## 四、与上月差异
- 哪些指标发生了显著变化
- 是否触发季节切换

## 五、风险点
- 可能改变当前判断的事件清单
```

**季节-仓位对应建议**（仅作参考，由你最终决定）：

| 季节 | 总仓位区间 | 风格倾向 |
|------|----------|---------|
| 冬中冬末 | 30-50% | 防守，高股息为主 |
| 春初 | 40-60% | 开始建仓困境反转、深度价值 |
| 春中 | 60-75% | 加仓低位成长股 |
| 春末夏初 | 75-90% | 主升浪初期，进攻 |
| 夏中 | 80-95% | 持有主线、轮动 |
| 夏末秋初 | 70-85% | 警惕，开始兑现 |
| 秋中秋末 | 40-70% | 持续减仓，留核心 |
| 冬初 | 20-40% | 防守，等待 |

### 3.2 风格判断 Agent

**核心问题**：当前是成长占优还是价值占优？

**输入指标**：
- 成长/价值指数比价及分位
- 大小盘风格（沪深 300 vs 中证 1000 比价）
- 利率环境（10 年期国债收益率方向）
- 行业景气离散度（哪些行业景气向上、哪些向下）

**输出**：
- 风格倾向（成长 / 平衡 / 价值）
- 建议的五个组之间的配置权重（默认 20% × 5，根据风格调整）
- 单板块最高暴露不超过 25%（参考柏基 Baillie Gifford 原则）
- 最低配置红线：任何板块 ≥ 3%（永不清零）

### 3.3 共识阶段 Agent

> 本 Agent 为 v2 新增，源自吴伟志访谈中关于"关注度-参与度"的分析框架。

**核心框架**：

| 阶段 | 关注度 | 参与度 | 含义 | 操作含义 |
|------|--------|--------|------|---------|
| 第一阶段 | 低 | 低 | 左侧机会 | 研究布局窗口，真正的"春天" |
| 第二阶段 | 高 | 低 | 过程中 | 安全区间，未来仍有买力支撑上行 |
| 第三阶段 | 高 | 高 | 过热 | 危险信号，准备减仓或严格不加仓 |

> 吴伟志原话："高关注度、高参与——高参与就说明大家都已经把钱投进去了，价格就一定会反映出来。这种时候往往是有危险了。如果大家都关注但都没有参与，说明未来还有上行的买力。"

**关注度代理指标**：
- 媒体提及频率（财联社/华尔街见闻/雪球热帖数）
- 百度指数 / 微信指数趋势
- 卖方研报覆盖数量变化
- 路演/电话会频率

**参与度代理指标**：
- 公募基金重仓持股变化（季报披露）
- 融资买入占比
- ETF 规模变化（行业 ETF）
- 北向资金行业配置变化
- 新发基金主题分布

**输出格式**（月度，按板块）：

```markdown
# 共识阶段判断报告 - YYYY 年 MM 月

## 各板块共识阶段

| 板块   | 关注度 | 参与度 | 阶段判断       | 相比上月 | 操作含义         |
|--------|--------|--------|---------------|---------|-----------------|
| 消费   | 低     | 低     | 第一阶段(左侧) | 无变化  | 研究布局窗口     |
| 医药   | 中     | 低     | 第一→二阶段过渡 | ↑      | 可以开始加仓     |
| 科技   | 高     | 中     | 第二阶段(过程中) | 无变化  | 持有，监控参与度 |
| 周期   | 低     | 低     | 第一阶段(左侧) | 无变化  | 研究布局窗口     |
| 高股息 | 高     | 高     | 第三阶段(过热) | ↑      | ⚠ 预警：减仓候选 |

## 预警
- [板块X] 已进入第三阶段（高关注+高参与），建议减仓或至少不加仓
- [板块Y] 参与度正在快速上升，关注是否接近第三阶段

## 关键观察
- 本月关注度/参与度变化最大的板块
- 餐桌指标：同行/朋友圈最近在聊什么
```

## 4. 量化辅助 Agent

> 本章为 v2 新增。吴伟志将量化定义为"工程机械"和"CT/核磁共振"——不替代主观判断，而是把模糊的直觉锚定到精确数据。
>
> 原话："消费估值很低了，但低到什么程度？成长很好、价值很差，他们之间的比价关系有不合理，但不知道到什么程度。这些对量化来说叫秒懂。"

### 4.1 定位

量化辅助 Agent **不做主观判断、不推荐买卖**。它只回答一个问题："现在是什么状态，精确到什么程度。"

它服务于两个上游：
- 服务投委会：提供市场整体和板块级别的精确状态数据
- 服务研究组：提供个股和行业级别的估值定位数据

### 4.2 核心输出

**A. 估值分位数据**
- 每个行业/个股的 PE/PB 当前在近 10 年的分位
- 横向：同行业公司估值对比排序
- 纵向：单个公司估值的历史分位

**B. 风格比价**
- 成长/价值指数比价，当前在历史分位的位置
- 大盘/小盘风格偏离度
- 偏离超过 1.5 个标准差时自动标记

**C. 行业比价**
- 跨行业的相对估值偏离度
- 哪些行业估值处于历史极端位置（10%以下或90%以上分位）

**D. 拥挤度指标**
- 机构持仓集中度（公募前50大重仓的行业分布）
- 融资买入占比（行业维度）
- 成交额占比（行业维度）

**E. 相关性监控**
- 持仓内个股的相关性矩阵
- 识别"以为分散但实际集中"的风险
- 行业暴露度计算

### 4.3 输出格式

```markdown

|----------|--------|---------|--------|---------|--------|

| 医药     | XX     | XX%     | XX     | XX%     | XX%    |
| 科技     | XX     | XX%     | XX     | XX%     | XX%    |
```

## 5. 建仓纪律协议

> 本章为 v2 新增。源自吴伟志访谈中明确描述的建仓纪律。
>
> 原话："左侧逐渐低吸，目标仓位的三成。吸完两到三成它还没有止跌，先停下来。然后跌不动了，我们会加到五成。符合我们的预期、上行确认了，我们就会把剩下的五成打满。"
>
> "如果你过早的 all in 进去，有可能你在底部割出来，它会全局错。"

### 5.1 三阶段建仓协议

```
建仓协议（每只股票独立执行）：

Phase 1 — 左侧试探（目标仓位的 ≤ 30%）
├── 前提条件：
│   ├── 深度研究报告已完成，GREAT 评分通过
│   ├── 估值进入买入区间（价值股：满足低PE/低PB/高股息率；成长股：PEG < 1 或未来市值法有足够空间）
│   └── 以买断公司标准衡量值得买
├── 操作：分批买入，不超过目标仓位的 30%
├── 停止条件：买满 30% 后若仍在下跌 → 暂停，不加仓
└── 心态：做好了"买了就跌"的准备，这是左侧的代价

Phase 2 — 底部确认（加至目标仓位的 50%）
├── 前提条件：
│   ├── Phase 1 已完成
│   ├── 价格企稳（不再创新低，成交萎缩，跌不动了）
│   └── 基本面逻辑未被破坏
├── 操作：加仓至目标仓位的 50%
└── 判断标准：不要求趋势反转，只要求"跌不动了"

Phase 3 — 右侧打满（加至目标仓位的 100%）
├── 前提条件：
│   ├── Phase 2 已完成
│   ├── 上行趋势确认（价格突破关键位，成交放大）
│   └── 基本面符合预期（业绩验证、催化剂兑现）
├── 操作：打满剩余仓位
└── 此时风险收益比最优：底部已确认，方向已明确

纪律红线：
✗ 绝不在 Phase 1 就 all-in
✗ Phase 1 买满后继续下跌，绝不"补仓摊薄"——等 Phase 2 条件满足再说
✗ 不跳过 Phase 直接打满
```

### 5.2 协调 Agent 的建仓纪律校验

协调 Agent 在每次调仓建议中，必须校验：

```
建仓纪律检查清单：
□ 该股票当前处于哪个 Phase？
□ 本次调仓后，是否超出当前 Phase 的仓位上限？
□ 是否满足进入下一个 Phase 的前提条件？
□ 如果 Phase 1 买入后仍在下跌，是否已暂停？

如果违反任何一条 → 标记为"⚠ 纪律违规"，要求基金经理确认
```

## 6. 股票池设计

**股票池是这套系统最重要的物理产出**。所有研究最终都要沉淀到这里。

### 6.1 分层结构

```
股票池
├── 核心持仓池（Core Pool）
│   └── 5-10 家，重仓，长期跟踪，深度研究
├── 卫星持仓池（Satellite Pool）
│   └── 10-20 家，中等仓位，季度调整
├── 观察池（Watch Pool）
│   └── 30-50 家，未持仓但持续跟踪，等待买点
└── 储备池（Reserve Pool）
    └── 50-100 家，初步覆盖，等待深入研究的候选
```

> 吴伟志的组合目标：50-80只股票，加权年化增长约18-20%，PE约14-15倍，股息率2-3%。核心持仓池+卫星持仓池对标这个范围。

### 6.2 每家公司的字段清单

**标签字段（必填）**：
- 股票代码 / 名称 / 上市地（A 股 / 港股 / 美股）
- 行业组归属（消费 / 医药 / 科技 / 周期 / 高股息）
- 公司类型（树 / 粮 / 菜）
- 商业模式（To C 消费品 / To B 工业品 / 平台 / 资源 / 金融 / 服务）
- 行业周期位置（春 / 夏 / 秋 / 冬，独立于大盘）
- 配置层级（核心 / 卫星 / 观察 / 储备）

**研究字段（深度报告同步更新）**：
- 主营业务拆分
- 产业链位置与竞争格局
- 护城河（品牌 / 技术 / 规模 / 网络 / 牌照 / 成本）
- GREAT 五维评分（每项 1-10 分 + 文字依据）
  - G (Growth): 成长速度
  - R (Replicable): 可复制性
  - P (Potential): 空间天花板
  - E (Excellent): 管理层
  - R (Return): 业绩确定性
- 财务画像：近 5 年营收/利润/现金流/ROE 趋势
- 估值带：合理 PE/PB 区间、目标价（保守/中性/乐观三档）
- 跟踪要点：3-5 个最关键的高频指标
- 卖出信号：3-5 个会让你减仓或清仓的信号

**v2 新增字段**：

- **建仓阶段**：Phase 0（未建仓） / Phase 1（左侧试探，≤30%） / Phase 2（底部确认，≤50%） / Phase 3（右侧打满，100%）
- **价格-价值反应状态**：没有反应 / 合理反应 / 过度反应（乐观） / 过度反应（悲观）
- **共识阶段**：第一阶段（低关注低参与） / 第二阶段（高关注低参与） / 第三阶段（高关注高参与）
- **买断公司检验**：一句话回答"如果以当前价格买下整家公司，你愿意吗？为什么？"

**状态字段（动态）**：
- 当前评级（强买 / 买入 / 增持 / 中性 / 减持 / 卖出）
- 上次评级变更时间和理由
- 仓位建议区间（在该评级下，组合中该给的比例）
- 关键催化剂时间表（业绩日、产品上市、政策窗口）
- 更新日志：每次跟踪的时间、内容、结论变化

### 6.3 入池标准

**储备池入池**（门槛低，广撒网）：
- 行业组研究员认为值得初步覆盖
- 至少完成商业模式 + 财务画像两项

**观察池入池**（必须有完整研究报告）：
- 完成 GREAT 五维评分
- 给出明确的目标价区间和买点
- 给出明确的跟踪要点和卖出信号
- 完成价格-价值反应状态判断
- 通过买断公司检验

**卫星持仓池入池**（具备买入条件）：
- 当前价格在合理估值带内或以下
- 行业周期位置在春/夏阶段，或冬末春初困境反转
- 投委会判断的市场季节不在秋末冬初
- 共识阶段不在第三阶段（高关注高参与）
- 建仓阶段从 Phase 0 进入 Phase 1

**核心持仓池入池**（最严格）：
- GREAT 评分各项不低于 7 分
- 类型为"树"（真正的长期成长）
- 至少经历过一轮完整季报验证
- 你能用一句话讲清楚为什么持有
- 建仓阶段至少进入 Phase 2

### 6.4 出池标准

任何一条满足就启动评级下调或出池流程：
- GREAT 评分某项跌破 5 分
- 触发预设的卖出信号
- 行业进入秋末冬初阶段
- 出现重大公司治理问题
- 估值进入历史极高分位（90% 以上）
- 共识阶段进入第三阶段 + 估值偏高（双重确认）
- 发现是价值陷阱（高股息不可持续）

### 7.2 逆向投资边界检查

> v2 新增。源自吴伟志访谈："逆向是逆行，不是逆知。"

协调 Agent 在整合分歧时，必须执行逆向检查：

```
逆向合理性检查：

✓ 合理的逆向（择时层面）：
  - 市场极度悲观时建议加仓
  - 市场极度亢奋时建议减仓
  - 某板块处于第一阶段（低关注低参与），研究组发现了价值

✗ 不合理的逆向（选股层面）：
  - 因为"大家都看好"某家公司所以故意不买
  - 因为某个行业是"共识方向"就回避
  - 为了"独立思考"而刻意与市场唱反调

判断标准：
- 共识方向上的好公司 → 看参与度，不看关注度
- 如果参与度低（机构没重仓），关注度高没关系——未来还有买力
- 只有当参与度已经很高时，才需要警惕
- 独立思考的结果跟市场一样 → 完全正常，不需要刻意不同

吴伟志原话："独立思考下来跟别人一样，为什么不可以呢？"
```

## 8. Agent Prompt 骨架

下面给出各类核心 Agent 的 Prompt 骨架，可以直接复制改用。

### 8.1 行业研究 Agent — 成长组通用骨架（消费/医药/科技）

```
你是一个专业的 [消费/医药/科技] 行业研究员，工作模式参考中欧瑞博。
你的角色是"内部卖方"——为基金经理和投委会提供研究素材，不做最终决策。

## 你的工作原则

1. 你只研究公司和行业，不判断市场整体季节，不给出"现在该不该买"
   这类全局判断。
2. 你的每一项研究都必须按 GREAT 五维框架展开：
   - Growth：当前和未来 3-5 年的成长速度
   - Replicable：商业模式是否可复制、可放大
   - Potential：行业天花板和公司空间（至少 5 倍空间才值得重仓）
   - Excellent：管理层评估（价值观、格局、视野、能力、文化）
   - Return：业绩确定性（不一定是利润，但要能兑现）
3. 你必须给每家公司打"树 / 粮 / 菜"的定性：
   - 树：长期复利、空间巨大的真成长，需要与企业共同成长
   - 粮：周期成长，春播秋收，节奏型机会
   - 菜：只有交易价值的题材股，原则上不进核心池
4. 你必须判断"价格-价值反应状态"：
   - 没有反应：基本面已变化，股价尚未体现
   - 合理反应：股价变动与基本面变化基本匹配
   - 过度反应（乐观方向）：股价涨幅超过基本面支撑
   - 过度反应（悲观方向）：股价跌幅超过负面因素影响
5. 你引用的任何数据都必须给出原始来源（链接或文件位置）。
6. 如果信息不足以判断，明确说"我不知道，需要补充 XX 数据"，
   不要硬给结论。报告末尾必须列出"还需进一步研究的问题"。
7. 以"买断公司"的心态评估：如果以当前价格买下整家公司，值不值？

## 估值红线（不可违反）

- 禁止：以"PE太高"为由否定高增长公司。PE 高低是价值股的语言，不是你的。
- 对有利润的成长股：必须用 PEG（PEG < 1 偏低估，> 1.5 偏贵）
- 对无利润或早期的新经济公司：必须用"未来市值法"
  （行业规模 × 公司份额 × 利润率 × 合理PE = 未来市值，对比当前市值）
- PS 可作为辅助参考，但不能作为唯一依据

吴伟志原话："如果一个成长股投资者跟我讲PE贵所以贵了，我就不跟他讨论。
就相当于你用一个尺子来称一个东西的重量一样。"

## 你的产出格式

[根据任务类型选择]：深度报告 / 跟踪报告 / 行业框架更新

## 当前任务
```

### 8.5 共识阶段 Agent（投委会，v2 新增）

```
你是中欧瑞博风格的市场共识分析师，专门负责判断各板块的
"关注度-参与度"所处阶段。

## 你的分析框架

吴伟志将市场共识分为三个阶段：
- 第一阶段：低关注度、低参与度 → 左侧机会
- 第二阶段：高关注度、低参与度 → 过程中，安全
- 第三阶段：高关注度、高参与度 → 过热，危险

核心原理：
- "逆向是逆行，不是逆知"——不要因为共识就回避，要看参与度
- 高关注不可怕，高参与才可怕
- 高参与意味着"大家都已经把钱投进去了，价格已经反映"
- 低参与意味着"未来还有上行的买力"

## 你的工作原则

1. 你对每个板块独立判断共识阶段，不判断具体公司。
2. 关注度通过媒体、搜索、研报数量等代理。
3. 参与度通过机构持仓、融资、ETF 规模等代理。
4. 当某板块进入第三阶段时，必须发出预警。
5. 你可以向量化辅助 Agent 请求拥挤度数据。

## 输入数据
[本期关注度和参与度数据]

## 输出格式
按"共识阶段判断报告"模板输出。

## 关键约束
- 不给个股建议
- 不判断市场整体季节（那是季节判断 Agent 的工作）
- "高关注+低参与"不是问题，不要因此就建议减仓
- 只有"高关注+高参与"才需要预警
```

```
   - 量化辅助 Agent 输出月度数据包
   - 五个组各自输出月度跟踪报告
2. 协调 Agent 整合为月度决策包（含板块温度地图 + 建仓纪律校验）
3. 你召开"月度投委会"：
   - 重新审视股票池分层
   - 审视各持仓的建仓阶段，是否该推进到下一 Phase
   - 决定本月调仓
   - 更新核心持仓的目标价
   - 启动 1-2 篇深度研究
   - 执行"海南检查"：有没有该转移资金的板块
```

### 10.4 季频流程（每季度半天）

```
季报披露后：
1. 五个组对池内公司全部财报点评
2. 全面更新 GREAT 评分
3. 全面更新价格-价值反应状态
4. 全面更新共识阶段判断
5. 调整目标价
6. 协调 Agent 启动季度复盘：
   - 本季所有决策的"当时怎么想—结果如何"
   - 错的归因（季节错 / 公司错 / 仓位错 / 执行错 / 纪律违规）
   - 对的归因（运气 / 实力）
   - 建仓纪律执行情况审计
7. 系统迭代：
   - Prompt 是否需要调整
   - 跟踪指标是否需要增减
   - 报告模板是否需要优化
   - 估值方法是否出现了跨体系混用
```

### 10.5 事件触发流程

```
当以下情况发生：
- 重大政策出台
- 黑天鹅事件（含地缘政治，如中东冲突、海峡事件）
- 持仓公司重大利空/利好
- 行业出现拐点信号
- 共识阶段 Agent 判定某板块从第二阶段跳到第三阶段

立即启动：
1. 相关 Agent 输出快速点评
2. 量化辅助 Agent 输出相关数据快照
3. 协调 Agent 整合
4. 你做应急决策（必要时不等到月会）
5. 记录到决策日志
```

## 11. 决策日志

这是整套系统最容易被忽略、但最有价值的产出物。

### 11.1 每次决策必须记录的字段

| 字段 | 说明 |
|------|------|
| 决策时间 | 精确到日期 |
| 决策类型 | 买入 / 加仓 / 减仓 / 清仓 / 仓位调整 / 池子调整 / Phase 推进 |
| 决策内容 | 具体动作 |
| 建仓阶段 | 本次操作涉及的 Phase 及是否合规 |
| 决策依据 | 当时的判断逻辑（季节判断 + 公司判断 + 风格判断 + 共识阶段 + 价格反应状态） |
| 当时数据 | 引用的关键数据快照（含量化辅助 Agent 提供的分位数据） |
| 预期结果 | 你预期会发生什么 |
| 复盘节点 | 什么时候回来检查（一般是 1 个月、3 个月、6 个月） |
| 实际结果 | 复盘时填入 |
| 偏差归因 | 季节错 / 公司错 / 仓位错 / 执行错 / 纪律违规 / 运气 |

### 11.2 决策日志的复盘价值

半年之后回看：
- 你会发现自己反复在哪类决策上犯错
- 你会发现自己的判断在哪类情况下更准
- 你会发现哪些 Agent 输出最有价值、哪些最容易误导
- 你会发现自己的真实能力圈在哪里
- 你会发现建仓纪律执行得好不好——是否有过早 all-in 或该加仓时犹豫

这是你的**个人版"致投资者的信"**，等价于吴伟志公开发表的那些季度信件。

## 12. 迭代路线图

不要试图一次性把所有东西搭起来。按下面的节奏推进。

### 阶段 1：单组验证（第 1-2 周）

**目标**：跑通最小闭环
- 挑你最熟悉的 1 个行业组（建议是你之前研究最多的）
- 搭建：行业框架文档 + 5-10 家公司档案 + Prompt 骨架 + 1 份深度报告
- 验证：Agent 能不能产出可用的研究（含价格反应状态和买断公司检验）
- 同时搭建量化辅助 Agent 的基础数据输出

**完成标志**：你能基于 Agent 的报告做出至少一次有把握的决策

### 阶段 2：双轨分离（第 3-4 周）

**目标**：把投委会从研究中独立出来
- 搭建季节判断 Agent + 共识阶段 Agent
- 完成第一份月度市场季节判断报告 + 共识阶段报告
- 验证物理隔离的有效性
- 验证建仓纪律协议的可执行性

**完成标志**：你在研究公司时不再被市场涨跌干扰

### 阶段 3：全员上线（第 2-3 个月）

**目标**：五个组全部跑起来
- 扩展到剩下四个行业组
- 加入风格判断、风险预警、协调 Agent
- 量化辅助 Agent 完善全部五类输出
- 跑通月度流程（含板块温度地图）
- 验证建仓纪律在多个标的上的执行

**完成标志**：连续 2 次月度投委会顺利召开，会议材料包可用

### 阶段 4：季度复盘与迭代（第 4-6 个月）

**目标**：完成第一次完整季度复盘，优化系统
- 整理过去 3 个月的所有决策
- 归因分析（含建仓纪律执行审计）
- 优化 Prompt、模板、流程
- 删除没用的环节，强化有用的
- 检查估值方法论是否出现跨体系混用

**完成标志**：你能清晰说出系统的强项和弱项

### 阶段 5：稳态运行（第 6 个月之后）

**目标**：进入稳定的"研究复利"状态
- 股票池积累到 80-150 家公司的完整档案
- 决策日志积累 50+ 条
- 系统能在任何市场环境下给出有依据的判断
- 建仓纪律成为"运动员本能"
- 你的认知和工具同步进步

## 13. 风险与边界

最后必须明确这套系统**不能做什么**，以及主要风险。

### 13.1 系统的天花板

1. **信息层面**：你拿不到机构终端、上市公司直接调研、产业专家网络。你的优势只能是"对公开信息的处理深度"，不可能是"信息领先"。这也是为什么报告中必须标注 70 分和 95 分深度的区分。

2. **判断层面**：Agent 在事实归纳上很强，在产业判断、管理层评估、拐点识别上仍然有限。吴伟志强调"必须成为用户"（他买特斯拉才理解新能源），这种体感认知 Agent 无法替代。

3. **执行层面**：交易纪律、情绪管理、仓位执行，Agent 帮不了你。系统设计得再好，真到大跌时手抖也没用。建仓纪律协议的存在就是为了减少"手抖"的空间。

4. **反人性层面**：Agent 在"反人性"方面可能反而有优势——它不会在底部恐惧、在顶部贪婪。但你作为最终决策者，仍然可能推翻 Agent 的建议。决策日志会记录这些推翻，复盘时检查是对是错。

### 13.2 主要风险

| 风险 | 表现 | 应对 |
|------|------|------|
| Agent 幻觉 | 编造数据、误读信息 | 强制溯源、抽检验证、关键决策双重确认 |
| 系统僵化 | 教条按框架走，忽视新情况 | 季度复盘必问"框架本身是否需要调整" |
| 过度依赖 | 不再独立思考 | 每月强制阅读年报、专访等"无 AI 中介"的原始材料 |
| 选股偏好沉淀 | Agent 反复推荐同一类公司 | 定期请 Agent 输出"与当前持仓最不同的候选" |
| 复盘流于形式 | 复盘只看赚了多少 | 强制每次决策填"归因"字段 |
| 估值跨体系混用 | 用PE否定成长股、用空间评价价值股 | Prompt 硬编码红线 + 季度审计 |
| 建仓纪律松弛 | Phase 1 就 all-in、该加仓时犹豫 | 协调 Agent 校验 + 决策日志记录 |

### 13.3 红线

任何时候不要做这几件事：
1. 让 Agent 直接执行交易
2. 跳过会议机制直接根据 Agent 输出下单
3. 因为 Agent 反复推荐就买入超出自己能力圈的公司
4. 让 Agent 帮你"找理由支持已经做的决定"
5. 把 Agent 的输出当成投资建议在外分享
6. 在 Phase 1 阶段就 all-in 任何标的
7. 因为"大家都看好"就逆向回避一只好公司

## 附录：吴伟志框架核心要点速查

### 四季理论
- 春：估值低、机构持仓低、情绪冷、基本面有改善早期信号——播种期
- 夏：估值合理偏高、情绪活跃、赚钱效应扩散——主升浪
- 秋：估值高、情绪亢奋、泡沫化——兑现期
- 冬：估值压缩、情绪极冷、流动性紧——防守期
- **关键**：天气 ≠ 季节，几连阳/几连跌都是天气
- **行业独立周期**：大盘在夏季，仍有行业在冬末春初；大盘在冬季，仍有行业进入春天

### 树/粮/菜
- 树：长期复利的真成长股，与企业共同成长
- 粮：周期成长，春播秋收
- 菜：只有交易价值的题材股

### GREAT 五维
- G (Growth)：成长性
- R (Replicable)：可复制性
- P (Potential)：空间天花板（至少 5 倍）
- E (Excellent)：管理层
- R (Return)：业绩兑现确定性

### 多小模型组大模型
- 消费小模型 + 科技小模型 + 周期小模型 + 价值/高股息小模型 + 医药小模型
- 每个小模型独立成熟，组合起来覆盖全市场

### 行业周期错配
- 大盘在夏季，仍有行业在冬末春初
- 大盘在冬季，仍有行业进入春天
- 所以"全天候"——任何季节都要有牌可打，前提是手里有股票池这副牌
- 永不清零：最差的板块也保留 3-5% 跟踪仓位

### 买断公司标准
- 不管成长还是价值，以买下整家公司的心态评估
- 价值股：PE两三倍就是两三年回本；股息率5%，即使不涨也不错
- 成长股：未来市值空间有多大，当前价格对应多少倍空间

### 估值方法论闭环
- 成长股：PEG（有利润）、未来市值法（无利润）、PS（辅助）
- 价值股：低PE + 低PB + 高股息率
- 周期股：PB、EV/EBITDA、重置成本
- **绝不混搭**：用PE否定成长股 = 用尺子称重量

### 价格-价值反应三种状态
- 没有反应：基本面变了，股价没动
- 合理反应：股价和基本面匹配
- 过度反应：股价走过头了（乐观或悲观方向）
- 投资赚钱的本质：在过度反应（悲观）时买入，在过度反应（乐观）时卖出

### 共识三阶段
- 第一阶段：低关注 + 低参与 → 左侧机会
- 第二阶段：高关注 + 低参与 → 安全，未来有买力
- 第三阶段：高关注 + 高参与 → 危险，过热
- **逆向是逆行，不是逆知**：择时可以逆向，选股不要抬杠

### 左侧建仓纪律
- Phase 1：左侧试探，≤ 目标仓位的 30%
- Phase 2：底部确认（跌不动了），加至 50%
- Phase 3：右侧确认（上行趋势），打满 100%
- **绝不在 Phase 1 就 all-in**

### "海南"策略
- 不在严冬板块"熬"，去春暖花开的地方
- 但"海南"也得早去——去晚了航班全没了
- 真正赚大钱的是左侧提前布局，不是追热点
- 资金从过热板块向低估板块转移

### 收益归因
- 收益来源于选股（50只股票加权年化增长18-20% + 股息2-3%）
- 仓位调整不增加收益，但降低波动、提升夏普比率
- "给你带来果实的，不是季节和气候，而是你种下去的庄稼"
